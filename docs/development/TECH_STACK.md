# StoryCraft 技术栈与依赖说明

## 最小依赖原则

StoryCraft 遵循**最小依赖原则**：

- 框架级运行时程序只依赖当前必需的第三方库。
- 每新增一个依赖，都必须有明确的功能理由，并在文档中记录。
- 优先使用标准库或项目内自建模块，避免引入不必要的抽象层或工具链。
- 开发依赖（如测试、静态检查、类型检查）与运行时依赖分离，不随生产环境安装。

## 依赖管理

所有依赖声明在 `pyproject.toml` 中：

- `project.dependencies`：运行时必须安装
- `project.optional-dependencies.dev`：开发时安装

安装方式：

```bash
# 仅安装运行依赖
pip install -e .

# 安装开发依赖
pip install -e ".[dev]"
```

## 新增依赖流程

如需新增运行时依赖，请按以下步骤进行：

1. 确认标准库或现有依赖无法满足需求。
2. 在 Issue 或 PR 中说明新增依赖的理由和替代方案。
3. 更新 `pyproject.toml` 中的依赖列表。
4. 更新本文档，记录新增依赖的用途和版本策略。
5. 确保新增的依赖不会引入与现有依赖的冲突。
