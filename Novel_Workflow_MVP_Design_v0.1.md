# Novel Workflow MVP Design v0.1

## 1. 项目目标

Novel Workflow 是一个用于辅助网文创作的 AI Workflow。

MVP 阶段不追求爆款，也不追求多 Agent 自动协作，而是验证：

> **是否能够通过固定 Workflow +
> RAG，实现从用户想法到可持续创作正文的稳定生产流程。**

MVP 关注的是流程是否稳定，而不是作品是否成功。

------------------------------------------------------------------------

# 2. MVP 原则

-   固定 Workflow，不使用复杂 Agent 协作。
-   所有步骤均输出标准化文档。
-   RAG 仅负责检索项目文档，不负责推理。
-   人工负责最终验证，不建立自动反馈闭环。
-   优先验证流程，再考虑自动优化。

------------------------------------------------------------------------

# 3. Workflow

``` text
User Idea
      │
      ▼
Story Synopsis
      │
      ▼
Outline
      │
      ▼
Scene Outline
      │
      ▼
Novel Writing
      │
      ▼
Update Project State
```

整个 Workflow 为单向流程，每一步仅负责一种能力。

------------------------------------------------------------------------

# 4. Workflow 说明

## Step1 Story Synopsis

输入：用户提供故事想法。

输出：

-   故事定位
-   世界观
-   主角定位
-   金手指
-   主线目标
-   支线方向
-   最终成长目标

仅建立故事骨架，不展开剧情。

------------------------------------------------------------------------

## Step2 Outline

根据 Story Synopsis 生成故事大纲。

每个 Story Node 包含：

-   当前目标
-   当前冲突
-   当前成长
-   当前收获
-   节点结束状态
-   下一节点入口

用于控制整本小说节奏。

------------------------------------------------------------------------

## Step3 Scene Outline

将 Story Node 拆分为多个 Scene。

每个 Scene 包含：

-   Scene 目标
-   冲突
-   爽点
-   钩子
-   伏笔
-   节奏
-   Scene 结束状态

Scene 为正文生成的直接依据。

------------------------------------------------------------------------

## Step4 Novel Writing

根据 Scene Outline 生成正文。

每章约 2000\~3000 字。

正文完成后，仅更新项目状态，不重新规划剧情。

------------------------------------------------------------------------

# 5. Project Assets

``` text
Project/
├── 00_Idea.md
├── 01_StorySynopsis.md
├── 02_Outline.md
├── 03_SceneOutline.md

├── 05_CharacterState.md
├── 06_Canon.md

    ├── Chapter001.md
    └── Chapter002.md
```

所有 Workflow 与 RAG 均围绕这些标准文档工作。

------------------------------------------------------------------------

# 6. Project State

## World State

维护：

-   势力
-   地图
-   世界规则
-   历史事件
-   世界变化

------------------------------------------------------------------------

## Character State

维护：

-   等级
-   技能
-   装备
-   财富
-   人际关系
-   当前目标
-   已完成事件

------------------------------------------------------------------------

## Canon

维护不可违反的设定：

-   世界规则
-   修炼规则
-   金手指限制
-   特殊设定

Canon 为最高优先级引用。

------------------------------------------------------------------------

# 7. RAG

MVP 阶段仅负责检索项目文档。

不负责：

-   自动学习
-   自动总结
-   自动评分
-   自动优化

唯一职责：

> 为当前 Workflow 提供稳定一致的上下文。

------------------------------------------------------------------------

# 8. MVP 验证目标

验证以下能力：

-   用户 Idea 能补全为完整故事概要。
-   Story Synopsis 能生成合理大纲。
-   大纲能拆分为可执行 Scene。
-   Scene 能稳定生成正文。
-   World State、Character State、Canon 能保持一致。

------------------------------------------------------------------------

# 9. MVP 不包含内容

-   多 Agent 协作
-   自动学习
-   自动知识库更新
-   流量分析
-   平台适配
-   自动评分
-   自动改稿
-   自动运营

------------------------------------------------------------------------

# 10. 后续规划

-   Workflow 为主
-   Agent 为增强

------------------------------------------------------------------------

# 11. Non-Goals（v0.1）

当前版本不会尝试解决：

-   如何写出爆款小说
-   如何自动分析作品数据
-   如何替代作者进行创意决策
-   如何实现完全自动化长篇连载
-   如何支持所有小说平台

MVP 只验证一件事：

> **Workflow + RAG 是否能够稳定完成从故事想法到正文创作的全过程。**
